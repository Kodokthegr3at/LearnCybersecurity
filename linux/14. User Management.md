## 1. 🏢 Overview — Why User Management Matters

### 🇮🇩 Bahasa Indonesia
**User management** adalah aspek fundamental dari administrasi sistem Linux. Administrator sering perlu:
- Membuat akun user baru untuk anggota tim
- Menambahkan user ke grup tertentu untuk kontrol akses
- Menjalankan perintah sebagai user berbeda untuk tugas yang membutuhkan privilege berbeda

Dari perspektif **cybersecurity**, memahami user management sangat penting untuk:
- Mengidentifikasi **privilege escalation** opportunities
- Menemukan **misconfigured permissions**
- Melakukan **auditing** dan troubleshooting sistem

### 🇬🇧 English
**User management** is a fundamental aspect of Linux system administration. Administrators frequently need to:
- Create new user accounts for team members
- Add users to specific groups to enforce access controls
- Execute commands as a different user for tasks requiring different privileges

From a **cybersecurity** perspective, understanding user management is critical for:
- Identifying **privilege escalation** opportunities
- Finding **misconfigured permissions**
- Performing system **auditing** and troubleshooting

### 🇯🇵 日本語
**ユーザー管理**はLinuxシステム管理の基本的な側面です。管理者が頻繁に行う作業：
- チームメンバーのための新しいユーザーアカウントの作成
- アクセス制御のためのユーザーへの特定グループへの追加
- 異なる権限が必要なタスクのための別ユーザーとしてのコマンド実行

**サイバーセキュリティ**の観点から、ユーザー管理の理解は以下のために重要です：
- **権限昇格**の機会の特定
- **誤った設定のパーミッション**の発見
- システムの**監査**とトラブルシューティング

### 🔐 The `/etc/shadow` File

```bash
# Regular user — CANNOT read /etc/shadow
$ cat /etc/shadow
cat: /etc/shadow: Permission denied

# With sudo — CAN read /etc/shadow
$ sudo cat /etc/shadow
root:<SNIP>:18395:0:99999:7:::
daemon:*:17737:0:99999:7:::
bin:*:17737:0:99999:7:::
```

> 🔴 `/etc/shadow` stores **encrypted password hashes** for all user accounts. It is readable/writable **only by root** to prevent unauthorized access to authentication data.

## 2. ⚡ `sudo` — Execute as Another User

### 🇮🇩 Bahasa Indonesia
`sudo` (superuser do) memungkinkan user yang diizinkan untuk **menjalankan perintah dengan hak akses user lain**, biasanya root. Ini adalah cara aman untuk melakukan tugas administratif **tanpa perlu login sebagai root** secara langsung — praktik terbaik keamanan sistem.

### 🇬🇧 English
`sudo` (superuser do) allows permitted users to **execute commands with the security privileges of another user**, typically root. This is the safe way to perform administrative tasks **without logging in as root** directly — a security best practice.

### 🇯🇵 日本語
`sudo`（superuser do）は許可されたユーザーが通常はrootなど**別のユーザーのセキュリティ権限でコマンドを実行**できます。これはrootとして直接ログインせずに管理タスクを実行する安全な方法です — セキュリティのベストプラクティスです。

```bash
# Run a single command as root
$ sudo cat /etc/shadow

# Run command as specific user
$ sudo -u alex cat /home/alex/secret.txt

# Open a root shell
$ sudo -s
$ sudo su

# Check what sudo privileges current user has
$ sudo -l

# Run with environment preserved
$ sudo -E command
```

> 💡 **Pentesting note:** `sudo -l` is one of the first commands to run after gaining access to a system — it reveals what commands you can run with elevated privileges without knowing the root password.

## 3. 🔄 `su` — Switch User

### 🇮🇩 Bahasa Indonesia
`su` (switch user) digunakan untuk **beralih ke sesi user lain** secara penuh. Berbeda dengan `sudo` yang hanya menjalankan satu perintah, `su` membuka **shell baru** sebagai user target. Membutuhkan password user tujuan (kecuali dijalankan oleh root).

### 🇬🇧 English
`su` (switch user) is used to **switch to another user's full session**. Unlike `sudo` which runs a single command, `su` opens a **new shell** as the target user. Requires the target user's password (unless run by root).

### 🇯🇵 日本語
`su`（switch user）は**別のユーザーのフルセッションに切り替え**るために使います。単一コマンドを実行する `sudo` とは異なり、`su` はターゲットユーザーとして**新しいシェルを開きます**。ターゲットユーザーのパスワードが必要です（rootが実行する場合を除く）。

```bash
# Switch to root (requires root password)
$ su
Password:
root@hostname#

# Switch to specific user
$ su alex
Password:
alex@hostname$

# Switch with full login environment (recommended)
$ su - alex

# Switch to root with login environment
$ su -
```

### `su` vs `sudo` — Key Difference

```
sudo command          →  runs ONE command as root, then back to your user
su                    →  opens a NEW SHELL as another user (stays until exit)
su -                  →  full login shell (loads target user's env variables)
sudo su               →  switch to root shell using sudo privileges
```

## 4. ➕ `useradd` — Create New User

### 🇮🇩 Bahasa Indonesia
`useradd` digunakan untuk **membuat akun user baru**. Saat user dibuat, sistem secara otomatis membuat direktori home, menetapkan UID, dan mengatur konfigurasi default dari `/etc/default/useradd`.

### 🇬🇧 English
`useradd` is used to **create a new user account**. When a user is created, the system automatically creates a home directory, assigns a UID, and applies default settings from `/etc/default/useradd`.

### 🇯🇵 日本語
`useradd` は**新しいユーザーアカウントを作成**するために使います。ユーザーが作成されると、システムは自動的にホームディレクトリを作成し、UIDを割り当て、`/etc/default/useradd` のデフォルト設定を適用します。

```bash
# Create a basic new user
$ sudo useradd alex

# Create user WITH home directory (-m) and default shell (-s)
$ sudo useradd -m -s /bin/bash alex

# Create user with specific UID (-u) and GID (-g)
$ sudo useradd -m -u 1500 -g developers alex

# Create user with comment/description (-c)
$ sudo useradd -m -c "Alex - Developer" alex

# Create user and add to supplementary groups (-G)
$ sudo useradd -m -G sudo,docker alex

# Create system user (no home dir, for services)
$ sudo useradd --system --no-create-home serviceaccount
```

| Flag | 🇮🇩 Fungsi | 🇬🇧 Function | 🇯🇵 機能 |
|------|-----------|-------------|---------|
| `-m` | Buat direktori home | Create home directory | ホームディレクトリを作成 |
| `-s` | Set default shell | Set default shell | デフォルトシェルを設定 |
| `-u` | Set UID spesifik | Set specific UID | 特定のUIDを設定 |
| `-g` | Set primary group | Set primary group | プライマリグループを設定 |
| `-G` | Tambah ke supplementary groups | Add to supplementary groups | 補助グループに追加 |
| `-c` | Tambah komentar/deskripsi | Add comment/description | コメント・説明を追加 |
| `-e` | Set tanggal kadaluarsa akun | Set account expiry date | アカウントの有効期限を設定 |

## 5. ❌ `userdel` — Delete User

### 🇮🇩 Bahasa Indonesia
`userdel` digunakan untuk **menghapus akun user** dari sistem. Gunakan flag `-r` untuk menghapus direktori home dan mail spool sekaligus. Tanpa `-r`, file user tetap tersisa di filesystem.

### 🇬🇧 English
`userdel` is used to **delete a user account** from the system. Use the `-r` flag to also remove the home directory and mail spool. Without `-r`, the user's files remain on the filesystem.

### 🇯🇵 日本語
`userdel` はシステムから**ユーザーアカウントを削除**するために使います。`-r` フラグを使うとホームディレクトリとメールスプールも同時に削除されます。`-r` なしではユーザーのファイルがファイルシステムに残ります。

```bash
# Delete user only (home directory remains)
$ sudo userdel alex

# Delete user AND their home directory + mail spool
$ sudo userdel -r alex

# Force delete even if user is logged in
$ sudo userdel -f alex
```

> ⚠️ Always use `userdel -r` carefully — it **permanently deletes** the user's home directory and all files within it.

## 6. ✏️ `usermod` — Modify User

### 🇮🇩 Bahasa Indonesia
`usermod` digunakan untuk **memodifikasi akun user yang sudah ada**. Ini adalah tool paling fleksibel untuk mengelola user — bisa mengubah username, shell, grup, direktori home, dan banyak lagi.

### 🇬🇧 English
`usermod` is used to **modify an existing user account**. It is the most flexible tool for managing users — can change username, shell, groups, home directory, and much more.

### 🇯🇵 日本語
`usermod` は**既存のユーザーアカウントを変更**するために使います。ユーザー名、シェル、グループ、ホームディレクトリなど様々な変更ができる最も柔軟なユーザー管理ツールです。

```bash
# Add user to a supplementary group
$ sudo usermod -aG docker alex
#                  ↑
#                  -a = append (IMPORTANT: without -a, existing groups are REPLACED)

# Add user to multiple groups
$ sudo usermod -aG sudo,docker,developers alex

# Change user's primary group
$ sudo usermod -g developers alex

# Change username
$ sudo usermod -l newname alex

# Change home directory
$ sudo usermod -d /new/home alex

# Change default shell
$ sudo usermod -s /bin/zsh alex

# Lock a user account
$ sudo usermod -L alex

# Unlock a user account
$ sudo usermod -U alex

# Set account expiry date
$ sudo usermod -e 2025-12-31 alex
```

> 🔴 **Critical:** Always use `-aG` (append + group), never just `-G` alone. Using `-G` without `-a` **removes** the user from all other groups first!

## 7. 👥 `addgroup` — Add Group

### 🇮🇩 Bahasa Indonesia
`addgroup` digunakan untuk **membuat grup baru** di sistem. Grup digunakan untuk mengelola akses ke file dan direktori secara kolektif — semua anggota grup berbagi permission yang sama.

### 🇬🇧 English
`addgroup` is used to **create a new group** on the system. Groups are used to manage file and directory access collectively — all group members share the same permissions.

### 🇯🇵 日本語
`addgroup` はシステムに**新しいグループを作成**するために使います。グループはファイルやディレクトリへのアクセスを一括管理するために使われます — グループメンバー全員が同じパーミッションを共有します。

```bash
# Create a new group
$ sudo addgroup developers

# Create group with specific GID
$ sudo addgroup --gid 1500 developers

# Add existing user to existing group
$ sudo addgroup alex developers
```

## 8. 🗑️ `delgroup` — Remove Group

### 🇮🇩 Bahasa Indonesia
`delgroup` digunakan untuk **menghapus grup dari sistem**. Grup tidak bisa dihapus jika masih ada user yang menggunakannya sebagai primary group.

### 🇬🇧 English
`delgroup` is used to **remove a group from the system**. A group cannot be deleted if it is still the primary group of any user.

### 🇯🇵 日本語
`delgroup` はシステムから**グループを削除**するために使います。まだそのグループをプライマリグループとしているユーザーがいる場合、削除できません。

```bash
# Remove a group
$ sudo delgroup developers

# Remove user from a group (without deleting the group)
$ sudo delgroup alex developers
```

## 9. 🔑 `passwd` — Change Password

### 🇮🇩 Bahasa Indonesia
`passwd` digunakan untuk **mengubah password user**. User biasa hanya bisa mengubah password mereka sendiri, sedangkan root bisa mengubah password siapa pun tanpa perlu mengetahui password lama.

### 🇬🇧 English
`passwd` is used to **change a user's password**. Regular users can only change their own password, while root can change anyone's password without knowing the old one.

### 🇯🇵 日本語
`passwd` はユーザーの**パスワードを変更**するために使います。一般ユーザーは自分のパスワードのみ変更できますが、rootは古いパスワードを知らなくても誰のパスワードでも変更できます。

```bash
# Change YOUR OWN password
$ passwd

# Change another user's password (requires root)
$ sudo passwd alex

# Lock a user's password (disable login)
$ sudo passwd -l alex

# Unlock a user's password
$ sudo passwd -u alex

# Force user to change password on next login
$ sudo passwd -e alex

# Check password status
$ sudo passwd -S alex
```

## 🔗 Practical Workflow Example

```bash
# Create new employee "alex" with full setup
$ sudo useradd -m -s /bin/bash -c "Alex - Developer" alex

# Set password
$ sudo passwd alex

# Add to necessary groups
$ sudo usermod -aG sudo,docker,developers alex

# Verify
$ id alex
uid=1001(alex) gid=1001(alex) groups=1001(alex),27(sudo),998(docker),1500(developers)

# Check home directory was created
$ ls -la /home/alex/

# Later: lock account if employee leaves
$ sudo passwd -l alex
$ sudo usermod -e 2025-01-01 alex
```

## 🧠 Quick Reference Cheatsheet

```bash
# ── CREATE ──────────────────────────────────────────────
sudo useradd -m -s /bin/bash username    # create user + home
sudo passwd username                      # set password
sudo addgroup groupname                   # create group
sudo usermod -aG groupname username       # add user to group

# ── READ / CHECK ────────────────────────────────────────
id username                               # show UID, GID, groups
whoami                                    # current user
groups username                           # show user's groups
cat /etc/passwd | grep username           # user info
sudo cat /etc/shadow | grep username      # password hash
sudo -l                                   # check sudo privileges

# ── MODIFY ──────────────────────────────────────────────
sudo usermod -aG group user               # add to group (SAFE)
sudo usermod -s /bin/zsh username         # change shell
sudo usermod -L username                  # lock account
sudo usermod -U username                  # unlock account
sudo passwd -e username                   # force pw change on login

# ── DELETE ──────────────────────────────────────────────
sudo userdel -r username                  # delete user + home
sudo delgroup groupname                   # delete group

# ── PRIVILEGE ESCALATION CHECK (pentesting) ─────────────
sudo -l                                   # what can I sudo?
cat /etc/passwd                           # list all users
cat /etc/group                            # list all groups
find / -perm -4000 2>/dev/null            # find SUID files
```
