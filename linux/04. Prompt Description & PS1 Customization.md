## 1. 🔤 Bash Prompt Basics

### 🇮🇩 Bahasa Indonesia
**Bash prompt** adalah baris teks yang muncul di terminal untuk memberi tahu kita bahwa sistem **siap menerima perintah**. Secara default, prompt menampilkan informasi penting seperti:
- **Username** — siapa yang sedang login
- **Hostname** — nama komputer/mesin
- **Current directory** — folder yang sedang aktif

Setelah prompt muncul, **kursor** (garis atau kotak berkedip) menunggu kita mengetik perintah.

### 🇬🇧 English
The **Bash prompt** is a line of text that appears in the terminal to let us know the system is **ready to receive commands**. By default, it displays important information such as:
- **Username** — who is currently logged in
- **Hostname** — the name of the computer/machine
- **Current directory** — the active folder

After the prompt appears, the **cursor** (blinking line or box) waits for us to type a command.

### 🇯🇵 日本語
**Bash プロンプト**はシステムが**コマンドを受け付ける準備ができた**ことを知らせるターミナルに表示されるテキスト行です。デフォルトで以下の重要な情報を表示します：
- **ユーザー名** — 現在ログインしているのは誰か
- **ホスト名** — コンピュータ/マシンの名前
- **現在のディレクトリ** — アクティブなフォルダ

プロンプトが表示された後、**カーソル**（点滅する線やボックス）がコマンドの入力を待ちます。

## 2. 📐 Prompt Structure

### 🇮🇩 Bahasa Indonesia
Format default bash prompt mengikuti pola yang konsisten dan mudah dipahami.

### 🇬🇧 English
The default bash prompt format follows a consistent, easy-to-understand pattern.

### 🇯🇵 日本語
デフォルトの Bash プロンプトの形式は一貫した分かりやすいパターンに従います。

---

### Default Format

```
<username>@<hostname><current working directory>$
```

### Anatomy

```
cry0l1t3  @  htb  [~]  $
    │     │   │    │    │
    │     │   │    │    └── $ = regular user  |  # = root user
    │     │   │    └─────── current directory (~ = home)
    │     │   └──────────── hostname (machine name)
    │     └──────────────── separator
    └────────────────────── username
```

### Real Examples

```bash
# Standard user in home directory
cry0l1t3@htb[~]$

# Standard user in a different directory
cry0l1t3@htb[/etc/ssh]$

# Root user
root@htb[/htb]#

# Minimal prompt (when PS1 not set — common in reverse shells)
$       ← unprivileged user
#       ← root / privileged
```

> 💡 **Tilde `~`** selalu merujuk ke **home directory** user yang sedang login. Contoh: `~` = `/home/cry0l1t3` untuk user biasa, atau `/root` untuk root.

## 3. 👤 User vs Root Prompt

### 🇮🇩 Bahasa Indonesia
Cara termudah untuk mengetahui apakah kita login sebagai **user biasa atau root** adalah dengan melihat **karakter terakhir** pada prompt:
- **`$`** = User biasa (unprivileged)
- **`#`** = Root / superuser (privileged)

Perbedaan ini sangat penting saat melakukan penetration testing — karakter `#` berarti kita sudah mendapatkan **root access**.

### 🇬🇧 English
The easiest way to know whether we are logged in as a **regular user or root** is by looking at the **last character** of the prompt:
- **`$`** = Regular user (unprivileged)
- **`#`** = Root / superuser (privileged)

This distinction is critical during penetration testing — the `#` character means we have achieved **root access**.

### 🇯🇵 日本語
**一般ユーザーかrootでログインしているか**を知る最も簡単な方法は、プロンプトの**最後の文字**を見ることです：
- **`$`** = 一般ユーザー（非特権）
- **`#`** = Root / スーパーユーザー（特権あり）

この違いはペネトレーションテスト中に非常に重要です — `#` 文字は**root アクセスを取得した**ことを意味します。

```bash
# Regular user prompt
cry0l1t3@htb[~]$  ← $ = normal user, limited privileges

# Root prompt
root@htb[/htb]#   ← # = root, FULL system access

# Minimal shell (e.g. after exploit / reverse shell)
$   ← still a user shell
#   ← got root! privilege escalation successful
```

> 🔴 **Pentesting note:** Saat menjalankan reverse shell pada target, kita mungkin hanya melihat `$` atau `#` tanpa username/hostname — ini karena variabel `PS1` belum di-set dengan benar di shell session tersebut.

## 4. ⚙️ PS1 Variable

### 🇮🇩 Bahasa Indonesia
**PS1 (Prompt String 1)** adalah variabel environment di Linux yang **mengontrol tampilan command prompt**. PS1 bertindak seperti template yang mendefinisikan teks yang muncul setiap kali sistem siap menerima perintah.

Dengan mengkustomisasi PS1, kita bisa menampilkan informasi tambahan yang berguna seperti:
- IP address mesin / target
- Tanggal dan waktu
- Status sukses/gagal perintah terakhir
- Full path direktori saat ini

Ini sangat berguna saat **penetration testing** karena membantu melacak aktivitas dan mendokumentasikan pekerjaan secara lebih efektif.

### 🇬🇧 English
**PS1 (Prompt String 1)** is an environment variable in Linux that **controls how the command prompt looks**. PS1 acts as a template that defines the text shown every time the system is ready for a command.

By customizing PS1, we can display additional useful information like:
- Machine / target IP address
- Date and time
- Success/failure status of the last command
- Full current directory path

This is especially useful during **penetration testing** as it helps track activity and document work more effectively.

### 🇯🇵 日本語
**PS1（Prompt String 1）** はLinuxの環境変数で、**コマンドプロンプトの表示方法を制御**します。PS1 はシステムがコマンドを受け付ける準備ができるたびに表示されるテキストを定義するテンプレートとして機能します。

PS1 をカスタマイズすることで以下の追加情報を表示できます：
- マシン/ターゲットのIPアドレス
- 日付と時刻
- 最後のコマンドの成功/失敗ステータス
- 現在のディレクトリのフルパス

これは**ペネトレーションテスト**中に活動の追跡と作業の文書化をより効果的に行うのに特に役立ちます。

```bash
# Check current PS1 value
$ echo $PS1
\[\u@\h \W\]\$

# Temporarily change PS1 in current session
$ PS1="[\u@\h \w] $ "

# Permanently change in ~/.bashrc
$ nano ~/.bashrc
# Add/modify PS1 line, then:
$ source ~/.bashrc
```

## 5. 🔣 PS1 Special Characters

### 🇮🇩 Bahasa Indonesia
PS1 menggunakan **karakter spesial** yang diawali dengan backslash `\` untuk menampilkan informasi dinamis secara otomatis.

### 🇬🇧 English
PS1 uses **special characters** prefixed with a backslash `\` to automatically display dynamic information.

### 🇯🇵 日本語
PS1 はバックスラッシュ `\` で始まる**特殊文字**を使って動的な情報を自動表示します。

---

| Character | 🇮🇩 Deskripsi | 🇬🇧 Description | 🇯🇵 説明 | Example Output |
|-----------|--------------|----------------|---------|----------------|
| `\u` | Username saat ini | Current username | 現在のユーザー名 | `cry0l1t3` |
| `\h` | Hostname (sampai titik pertama) | Hostname (up to first dot) | ホスト名（最初のドットまで） | `htb` |
| `\H` | Full hostname | Full hostname | フルホスト名 | `htb.local` |
| `\w` | Full path direktori saat ini | Full path of current directory | 現在のディレクトリのフルパス | `/home/user/docs` |
| `\W` | Nama folder saat ini saja | Current directory name only | 現在のディレクトリ名のみ | `docs` |
| `\d` | Tanggal (Mon Feb 6) | Date (Mon Feb 6) | 日付 | `Mon Feb  6` |
| `\D{%Y-%m-%d}` | Tanggal format YYYY-MM-DD | Date in YYYY-MM-DD format | YYYY-MM-DD 形式の日付 | `2025-06-01` |
| `\t` | Waktu 24 jam (HH:MM:SS) | Time 24-hour (HH:MM:SS) | 24時間形式の時刻 | `14:32:10` |
| `\T` | Waktu 12 jam (HH:MM:SS) | Time 12-hour (HH:MM:SS) | 12時間形式の時刻 | `02:32:10` |
| `\@` | Waktu 12 jam (AM/PM) | Time 12-hour (AM/PM) | 12時間形式（AM/PM） | `02:32 PM` |
| `\s` | Nama shell | Name of the shell | シェルの名前 | `bash` |
| `\j` | Jumlah background jobs | Number of background jobs | バックグラウンドジョブ数 | `2` |
| `\n` | Baris baru (newline) | Newline | 改行 | ↵ |
| `\$` | `$` untuk user, `#` untuk root | `$` for user, `#` for root | ユーザーは`$`、rootは`#` | `$` or `#` |

## 6. 🎨 Prompt Customization Examples

### 🇮🇩 Bahasa Indonesia
PS1 dikonfigurasi di file `~/.bashrc`. Perubahan permanen harus ditambahkan di file ini lalu di-source agar langsung berlaku.

### 🇬🇧 English
PS1 is configured in the `~/.bashrc` file. Permanent changes must be added to this file and sourced to take effect immediately.

### 🇯🇵 日本語
PS1 は `~/.bashrc` ファイルで設定します。永続的な変更はこのファイルに追加し、すぐに反映させるためにソースする必要があります。

---

### Preset Examples

```bash
# ── MINIMAL (clean and simple) ──────────────────────
PS1="[\u@\h \W]\$ "
# Output: [cry0l1t3@htb ~]$

# ── WITH FULL PATH ───────────────────────────────────
PS1="[\u@\h \w]\$ "
# Output: [cry0l1t3@htb /home/cry0l1t3/documents]$

# ── WITH DATE & TIME ─────────────────────────────────
PS1="[\D{%Y-%m-%d} \t] \u@\h:\w\$ "
# Output: [2025-06-01 14:32:10] cry0l1t3@htb:~$

# ── PENTESTING PROMPT (with IP tracking) ─────────────
PS1="[\u@\h - $(hostname -I | awk '{print $1}')] \w\$ "
# Output: [cry0l1t3@htb - 10.10.14.5] /home/cry0l1t3$

# ── WITH COLOR (green user, blue path) ───────────────
PS1="\[\033[01;32m\]\u@\h\[\033[00m\]:\[\033[01;34m\]\w\[\033[00m\]\$ "
# Output: cry0l1t3@htb:/home/cry0l1t3$  (colored)
```

### Color Codes Reference

```bash
# Format: \[\033[<code>m\]...\[\033[00m\]
#                    └── color code

# Text Colors
\[\033[30m\]   # Black
\[\033[31m\]   # Red
\[\033[32m\]   # Green
\[\033[33m\]   # Yellow
\[\033[34m\]   # Blue
\[\033[35m\]   # Purple
\[\033[36m\]   # Cyan
\[\033[37m\]   # White

# Bold versions
\[\033[01;32m\]  # Bold Green
\[\033[01;34m\]  # Bold Blue
\[\033[01;31m\]  # Bold Red

# Reset
\[\033[00m\]   # Reset to default
```

### How to Apply Permanently

```bash
# 1. Open .bashrc
$ nano ~/.bashrc

# 2. Find or add the PS1 line at the bottom:
PS1="[\D{%Y-%m-%d} \t] \u@\h:\w\$ "

# 3. Save and apply without restarting terminal
$ source ~/.bashrc
# or
$ . ~/.bashrc
```

### Pentesting Workflow — Recording Commands

```bash
# Record all commands with timestamps using script
$ script -a -t 2>~/pentest_timing.txt ~/pentest_log.txt

# View bash history with timestamps
$ export HISTTIMEFORMAT="%F %T "
$ history

# Bash history file location
$ cat ~/.bash_history
```

> 💡 Setting a custom PS1 with date/time + logging with `script` is a **best practice during pentests** — it automatically documents every command with a timestamp for reporting purposes.

## 🧠 Quick Reference Cheatsheet

```bash
# ── CHECK CURRENT PROMPT ─────────────────────────────
echo $PS1                    # show current PS1 value
echo $0                      # show current shell

# ── IDENTIFY USER LEVEL ──────────────────────────────
whoami                       # current username
id                           # UID, GID, groups
# $ at end = regular user
# # at end = root

# ── TEMPORARY CHANGE ─────────────────────────────────
PS1="[\u@\h \w]\$ "         # change for current session only

# ── PERMANENT CHANGE ─────────────────────────────────
echo 'PS1="[\u@\h \w]\$ "' >> ~/.bashrc
source ~/.bashrc

# ── COMMON PS1 PRESETS ───────────────────────────────
# Minimal:     PS1="[\u@\h \W]\$ "
# Full path:   PS1="[\u@\h \w]\$ "
# With time:   PS1="[\t] \u@\h:\w\$ "
# Pentesting:  PS1="[\D{%Y-%m-%d} \t] \u@\h \w\$ "

# ── RECORD COMMANDS (pentest documentation) ──────────
script -a ~/session.log      # start recording
exit                         # stop recording
export HISTTIMEFORMAT="%F %T "  # add timestamps to history
```
